Hoisting is JavaScript’s default behavior of reading all defined variables or function declarations at the top of the scope before any other code execution occurs. One of the benefits of hoisting in JavaScript is that it allows us to call functions before they appear in code, therefore abstract out our code some.

Also, it’s key to note that JavaScript only hoists function declarations, not initializations or arrow functions.

No matter where a function is declared, it is moved to the top of its scope regardless if it has a global or local scope. So one could say, the main advantage of using hoisting is that functions don’t have to be declared in a specific order in order to work properly.

Hoisting is not necessarily considered all that great and maybe even bad practice. Let’s say you have many declared variables and function declarations all being hoisted, these will all get read in the browser before any code is executed. All of this hoisting in turn will likely slow down the application's runtime quite dramatically.
